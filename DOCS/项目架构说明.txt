================================================================================
                          1. 架构概览
================================================================================

本项目采用7层标准架构设计，从上至下职责清晰，严格分层解耦，
层与层之间单向依赖，禁止跨层调用。

架构分层图:

┌─────────────────────────────────────────────────────────────────────────────┐
│ 第一层: 前端交互层 (Presentation Layer)                                      │
│ 负责小游戏的前端界面渲染、用户操作交互、前端事件响应                          │
├─────────────────────────────────────────────────────────────────────────────┤
│ 第二层: 统一网关层 (Gateway Layer)                                           │
│ 基于FastAPI，统一处理所有前端请求，实现接口鉴权、登录状态校验、               │
│ 请求限流、请求参数校验、跨域处理等网关通用能力                                │
├─────────────────────────────────────────────────────────────────────────────┤
│ 第三层: 业务工具通用层 (Common Layer)                                        │
│ 封装全项目通用的工具方法、业务通用逻辑                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│ 第四层: AI Agent 核心调度层 (AI Agent Layer)                                 │
│ 小游戏AI核心调度中心，负责AI行为决策、任务分配、流程编排                      │
├─────────────────────────────────────────────────────────────────────────────┤
│ 第五层: LLM 大模型接入层 (LLM Layer)                                         │
│ 统一封装与各类大模型的对接逻辑，屏蔽底层模型差异                              │
├─────────────────────────────────────────────────────────────────────────────┤
│ 第六层: 数据模型层 (Model Layer)                                             │
│ 定义项目全量数据结构、请求/响应模型、业务实体模型(基于Pydantic)               │
├─────────────────────────────────────────────────────────────────────────────┤
│ 第七层: 数据持久层 (Persistence Layer)                                       │
│ 负责数据的存储、读取、更新、删除，对接数据库/文件系统                         │
└─────────────────────────────────────────────────────────────────────────────┘

================================================================================
                          2. 分层说明
================================================================================

第一层: 前端交互层
------------------
目录: /client
职责:
  - 小游戏前端界面渲染
  - 用户操作交互处理
  - 前端事件响应
  - 调用后端API

核心文件:
  - game.js: 游戏入口
  - client/pages/login/: 登录页面
  - client/pages/main/: 主页面
  - client/utils/api.js: API请求封装
  - client/utils/auth.js: 认证管理

第二层: 统一网关层
------------------
目录: /backend/app/api
职责:
  - 接收所有前端请求
  - 接口鉴权(JWT验证)
  - 登录状态校验
  - 请求参数校验
  - 跨域处理
  - 路由分发

核心文件:
  - backend/app/main.py: FastAPI应用入口
  - backend/app/api/auth.py: 认证相关接口
  - backend/app/api/user.py: 用户相关接口

第三层: 业务工具通用层
----------------------
目录: /backend/app/utils
职责:
  - 工具函数封装
  - 通用校验逻辑
  - 格式转换
  - 异常处理
  - 日志工具

第四层: AI Agent 核心调度层
----------------------------
目录: /backend/app/ai_agent
职责:
  - AI行为决策
  - 任务分配
  - 流程编排
  - 协调下层模块完成AI相关业务

第五层: LLM 大模型接入层
------------------------
目录: /backend/app/llm
职责:
  - 统一封装大模型对接逻辑
  - 接口调用
  - 参数封装
  - 响应解析
  - 异常重试
  - 屏蔽底层模型差异

第六层: 数据模型层
------------------
目录: /backend/app/models
职责:
  - 定义全量数据结构
  - 请求/响应模型
  - 业务实体模型
  - 基于Pydantic实现数据校验与类型约束

核心文件:
  - backend/app/models/user.py: 用户模型
  - backend/app/models/character.py: 角色模型
  - backend/app/models/auth.py: 认证相关模型

第七层: 数据持久层
------------------
目录: /backend/app/db
职责:
  - 数据存储
  - 数据读取
  - 数据更新
  - 数据删除
  - 对接数据库/文件系统

核心文件:
  - backend/app/db/database.py: 数据库操作类

                          4. 数据流向
================================================================================

用户登录流程:
┌──────────┐    wx.login()    ┌──────────┐    code      ┌──────────┐
│  小游戏   │ ───────────────> │  前端层   │ ──────────> │  网关层   │
│  客户端   │                  │          │             │  (FastAPI)│
└──────────┘                  └──────────┘             └────┬─────┘
                                                            │
                                                            ▼
┌──────────┐    token/user_id  ┌──────────┐    openid    ┌──────────┐
│  小游戏   │ <───────────────  │  前端层   │ <─────────  │ 微信服务  │
│  客户端   │                  │          │             │          │
└──────────┘                  └──────────┘             └──────────┘

API请求流程:
┌──────────┐    HTTP Request   ┌──────────┐    鉴权      ┌──────────┐
│  小游戏   │ ────────────────> │  网关层   │ ─────────> │  服务层   │
│  客户端   │  + Bearer Token   │          │            │          │
└──────────┘                   └──────────┘            └────┬─────┘
                                                            │
                                                            ▼
┌──────────┐    JSON Response  ┌──────────┐    数据      ┌──────────┐
│  小游戏   │ <──────────────── │  网关层   │ <────────  │ 持久层    │
│  客户端   │                  │          │            │          │
└──────────┘                  └──────────┘            └──────────┘

================================================================================
                          5. 技术栈
================================================================================

后端技术栈:
-----------
- Python 3.9+
- FastAPI 0.104.1 (Web框架)
- Pydantic 2.5.0 (数据验证)
- python-jose 3.3.0 (JWT认证)
- httpx 0.25.2 (HTTP客户端)
- Uvicorn 0.24.0 (ASGI服务器)

前端技术栈:
-----------
- 微信小游戏框架
- JavaScript ES6+
- WXML (微信标记语言)
- WXSS (微信样式语言)

数据存储:
---------
- JSON文件存储 (当前实现)
- 可扩展为: MySQL / MongoDB / Redis
