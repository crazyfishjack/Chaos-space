## 一、架构概述

本项目采用**微信云数据库**作为唯一数据存储方案，完全替代原有的本地JSON文件存储。

### 1.1 架构演进

| 版本 | 存储方案 | 状态 |
|------|----------|------|
| v1.0 | 本地JSON文件存储 | 已废弃 |
| v2.0 | 微信云数据库 | 当前使用 |

### 1.2 技术选型理由

- **微信云数据库**: 与微信生态深度集成，支持云托管免鉴权调用
- **自动扩缩容**: 根据负载自动调整，支持1000+日活用户
- **数据安全**: 腾讯云基础设施保障，支持数据备份与恢复
- **开发便捷**: 本地开发可通过HTTP API直接访问云端数据库

## 三、数据库实现层 (app/db/)

### 3.1 模块结构

```
app/db/
├── __init__.py          # 数据库模块初始化，导出统一接口
└── wx_cloud_db.py       # 微信云数据库实现
```

### 3.2 核心类说明

#### `WxCloudDatabase` (异步实现)

**功能**: 微信云数据库异步操作类，通过HTTP API访问云数据库

**核心方法**:

##### `_get_access_token()`
- **函数签名**: `async def _get_access_token(self) -> str`
- **功能**: 获取微信云开发access_token
- **流程**:
  1. 优先使用环境变量配置的access_token
  2. 如未配置，使用AppID和AppSecret调用微信接口获取
- **返回**: access_token字符串

##### `_database_query(collection, query)`
- **函数签名**: `async def _database_query(self, collection: str, query: str) -> List[Dict]`
- **功能**: 执行数据库查询操作
- **参数**:
  - `collection`: 集合名称（users/characters）
  - `query`: 数据库查询语句
- **返回**: 查询结果列表（已解析为字典）
- **重要说明**:
  - 微信云数据库返回的 data 字段是 JSON 字符串数组
  - 本方法会自动解析 JSON 字符串为 Python 字典
  - 统一使用 where() 查询，避免 doc() 返回格式不一致问题

##### `_database_add(collection, data)`
- **函数签名**: `async def _database_add(self, collection: str, data: Dict) -> str`
- **功能**: 向集合中添加文档
- **参数**:
  - `collection`: 集合名称
  - `data`: 文档数据字典
- **返回**: 新创建文档的ID

##### `_database_update(collection, doc_id, data)`
- **函数签名**: `async def _database_update(self, collection: str, doc_id: str, data: Dict) -> bool`
- **功能**: 更新指定文档
- **参数**:
  - `collection`: 集合名称
  - `doc_id`: 文档ID
  - `data`: 更新数据字典
- **返回**: 是否更新成功

##### `_generate_unique_uid()`
- **函数签名**: `async def _generate_unique_uid(self) -> str`
- **功能**: 生成唯一的8位数字UID
- **流程**:
  1. 随机生成8位数字（首位不为0）
  2. 查询数据库检查UID是否已存在
  3. 如存在则重新生成，最多尝试10次
  4. 返回唯一UID
- **返回**: 8位数字字符串

#### `WxCloudDatabaseSync` (同步包装 - 已废弃)

**状态**: 已废弃，不再使用

**原因**: 在 FastAPI 异步环境下会导致事件循环冲突，抛出 `Cannot run the event loop while another loop is running` 错误

**替代方案**: 直接使用 `WxCloudDatabase` 异步类，所有调用使用 `await`

### 3.3 用户相关操作

#### `get_user_by_openid(openid)`
- **功能**: 通过微信openid查询用户
- **实现**: 使用 `where({openid: 'xxx'}).get()` 查询users集合
- **返回**: User对象或None
- **注意**: 统一使用 where() 查询，避免 doc() 返回格式不一致问题

#### `get_user_by_id(user_id)`
- **功能**: 通过用户ID查询用户
- **实现**: 使用 `where({id: 'xxx'}).get()` 查询users集合
- **返回**: User对象或None
- **注意**: 使用 id 字段查询（与 _id 字段值相同），保持与 where() 查询格式一致

#### `create_user(user_create)`
- **功能**: 创建新用户
- **流程**:
  1. 生成UUID作为用户ID
  2. 构建用户数据字典（包含_id、id、openid、unionid等字段）
  3. 调用_database_add插入users集合
  4. 返回User对象

#### `update_user_login_time(user_id)`
- **功能**: 更新用户最后登录时间
- **实现**: 更新last_login_at和updated_at字段

### 3.4 角色相关操作

#### `get_character_by_user_id(user_id)`
- **功能**: 查询用户的角色信息
- **实现**: 查询characters集合，匹配user_id字段
- **返回**: Character对象或None

#### `create_character(user_id, char_create)`
- **功能**: 为用户创建角色
- **流程**:
  1. 调用 `_generate_unique_uid()` 生成唯一8位UID
  2. 自动生成昵称"意识体{uid}"
  3. 构建角色数据（包含_id、id、user_id、nickname、uid等）
  4. 调用_database_add插入characters集合
  5. 返回Character对象
- **重要说明**: 
  - 角色昵称和UID由后端自动生成
  - 无需前端传入任何参数

#### `has_character(user_id)`
- **功能**: 检查用户是否已有角色
- **实现**: 调用get_character_by_user_id判断返回值

## 四、数据集合结构

### 4.1 users 集合

存储用户基础信息

| 字段名 | 类型 | 说明 |
|--------|------|------|
| _id | string | 文档主键，与用户id一致 |
| id | string | 用户唯一标识（UUID） |
| openid | string | 微信用户openid |
| unionid | string | 微信用户unionid（可选） |
| nickname | string | 用户昵称 |
| avatar_url | string | 头像URL |
| created_at | string | 创建时间（ISO格式） |
| updated_at | string | 更新时间（ISO格式） |
| last_login_at | string | 最后登录时间 |
| is_active | boolean | 是否激活 |

**索引建议**:
```json
{
  "openid": 1
}
```

### 4.2 characters 集合

存储游戏角色信息

| 字段名 | 类型 | 说明 |
|--------|------|------|
| _id | string | 文档主键，与角色id一致 |
| id | string | 角色唯一标识（UUID） |
| user_id | string | 所属用户ID |
| nickname | string | 角色昵称（默认格式：意识体{uid}） |
| uid | string | 角色唯一编号，8位数字 |
| created_at | string | 创建时间（ISO格式） |
| updated_at | string | 更新时间（ISO格式） |

**索引建议**:
```json
{
  "user_id": 1
}
```

**字段变更记录**:
- v2.1.0: 
  - 新增 `uid` 字段（8位唯一数字编号）
  - `name` 字段更名为 `nickname`
  - 移除 `level` 字段（不再需要）
  - 移除 `exp` 字段（不再需要）

## 六、性能优化建议

### 6.1 索引优化

已在微信云数据库控制台为以下字段创建索引：
- users.openid - 加速登录查询
- characters.user_id - 加速角色查询

### 6.2 并发处理

- 1000日活场景下，微信云数据库默认可承载
- 如需更高并发，可考虑：
  1. 升级云开发实例规格
  2. 添加Redis缓存层
  3. 使用云函数批量操作

### 6.3 数据备份

建议在微信云数据库控制台开启：
- 自动备份策略
- 数据回档能力

## 七、故障排查

### 7.1 常见错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| 获取access_token失败 | AppID/AppSecret错误 | 检查配置 |
| 数据库查询失败 | 集合不存在 | 在控制台创建集合 |
| 权限不足 | 数据库权限配置 | 检查云数据库安全规则 |
| UID生成失败 | 并发冲突 | 增加重试次数或使用UUID备用方案 |
