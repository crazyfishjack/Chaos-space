## 一、API 接口层 (app/api/)

### 1.1 认证接口 (auth.py)

#### `wechat_login()`
- **函数签名**: `async def wechat_login(login_data, x_wx_openid, x_wx_unionid, x_wx_appid)`
- **路由**: `POST /api/v1/auth/wx-login`
- **功能**: 微信用户登录入口，支持两种登录方式
  - 云托管免鉴权方式（优先）：从 Header 读取微信用户信息
  - 传统 code 方式：使用微信临时凭证换取用户信息
- **流程**:
  1. 优先检查 Header 中的 X-WX-OPENID（云托管方式）
  2. 如无 Header，则使用 code 调用微信 API 换取 openid
  3. 调用 UserService 获取或创建用户
  4. 生成 JWT 访问令牌
  5. 检查用户是否已创建角色
  6. 返回登录结果

## 二、数据模型层 (app/models/)

### 2.1 认证模型 (auth.py)

#### `WXLoginRequest`
- **类型**: Pydantic BaseModel
- **功能**: 微信登录请求参数模型
- **字段**:
  - `code`: Optional[str] - 微信登录临时凭证（云托管方式可不传）

#### `WXLoginResponse`
- **类型**: Pydantic BaseModel
- **功能**: 微信登录响应数据模型
- **字段**:
  - `access_token`: str - JWT 访问令牌
  - `token_type`: str - 令牌类型，固定为 "bearer"
  - `expires_in`: int - 令牌过期时间（秒），默认 7200
  - `user_id`: str - 用户唯一标识
  - `has_character`: bool - 用户是否已创建游戏角色

#### `TokenPayload`
- **类型**: Pydantic BaseModel
- **功能**: JWT 令牌载荷模型，定义令牌中包含的数据
- **字段**:
  - `sub`: str - 用户 ID（subject）
  - `openid`: str - 微信用户 openid
  - `exp`: Optional[int] - 过期时间戳

### 2.2 用户模型 (user.py)

#### `UserBase`
- **类型**: Pydantic BaseModel
- **功能**: 用户基础属性模型
- **字段**:
  - `openid`: str - 微信用户唯一标识
  - `unionid`: Optional[str] - 微信 unionid
  - `nickname`: Optional[str] - 用户昵称
  - `avatar_url`: Optional[str] - 用户头像 URL

#### `UserCreate`
- **类型**: Pydantic BaseModel，继承 UserBase
- **功能**: 创建用户请求模型

#### `User`
- **类型**: Pydantic BaseModel，继承 UserBase
- **功能**: 用户完整数据模型
- **字段**:
  - `id`: str - 用户 ID
  - `created_at`: datetime - 创建时间
  - `updated_at`: datetime - 更新时间
  - `last_login_at`: Optional[datetime] - 最后登录时间
  - `is_active`: bool - 是否激活

#### `UserResponse`
- **类型**: Pydantic BaseModel
- **功能**: 用户响应数据模型
- **字段**:
  - `id`: str - 用户 ID
  - `nickname`: Optional[str] - 用户昵称
  - `avatar_url`: Optional[str] - 头像 URL
  - `created_at`: datetime - 创建时间
  - `has_character`: bool - 是否已创建角色

### 2.3 角色模型 (character.py)

#### `CharacterBase`
- **类型**: Pydantic BaseModel
- **功能**: 角色基础属性模型
- **字段**:
  - `name`: str - 角色名称（1-20 字符）

#### `CharacterCreate`
- **类型**: Pydantic BaseModel，继承 CharacterBase
- **功能**: 创建角色请求模型

#### `Character`
- **类型**: Pydantic BaseModel，继承 CharacterBase
- **功能**: 角色完整数据模型
- **字段**:
  - `id`: str - 角色 ID
  - `user_id`: str - 所属用户 ID
  - `level`: int - 角色等级，默认 1
  - `exp`: int - 角色经验，默认 0
  - `created_at`: datetime - 创建时间
  - `updated_at`: datetime - 更新时间

#### `CharacterResponse`
- **类型**: Pydantic BaseModel
- **功能**: 角色响应数据模型
- **字段**:
  - `id`: str - 角色 ID
  - `name`: str - 角色名称
  - `level`: int - 角色等级
  - `exp`: int - 角色经验
  - `created_at`: datetime - 创建时间

## 三、服务层 (app/services/)

### 3.1 微信服务 (wechat_service.py)

#### `WechatService` (类)
- **功能**: 微信服务类，封装与微信服务器的交互
- **类属性**:
  - `WX_API_BASE`: str - 微信 API 基础地址

##### `WechatService.code2session()`
- **函数签名**: `async def code2session(cls, code: str) -> Dict[str, Any]`
- **功能**: 微信登录凭证校验，通过 code 换取 openid 和 session_key
- **参数**:
  - `code`: str - 微信登录临时凭证
- **流程**:
  1. 构建请求参数（appid, secret, js_code, grant_type）
  2. 发送 HTTP GET 请求到微信 API
  3. 检查返回结果中的错误码
  4. 返回包含 openid、session_key 等信息的字典

##### `WechatService.get_access_token()`
- **函数签名**: `async def get_access_token(cls) -> str`
- **功能**: 获取微信小程序全局 access_token
- **流程**:
  1. 构建请求参数（grant_type, appid, secret）
  2. 发送 HTTP GET 请求到微信 API
  3. 返回 access_token 字符串

### 3.2 用户服务 (user_service.py)

#### `UserService` (类)
- **功能**: 用户服务类，处理用户业务逻辑

##### `UserService.get_or_create_user()`
- **函数签名**: `def get_or_create_user(cls, openid: str, unionid: Optional[str] = None) -> User`
- **功能**: 获取或创建用户
- **参数**:
  - `openid`: str - 微信 openid
  - `unionid`: Optional[str] - 微信 unionid
- **流程**:
  1. 通过 openid 查询用户是否存在
  2. 如存在，更新最后登录时间并返回用户
  3. 如不存在，创建新用户并返回

##### `UserService.get_user_by_id()`
- **函数签名**: `def get_user_by_id(cls, user_id: str) -> Optional[User]`
- **功能**: 通过用户 ID 获取用户信息

##### `UserService.has_character()`
- **函数签名**: `def has_character(cls, user_id: str) -> bool`
- **功能**: 检查用户是否已创建角色

## 五、核心工具层 (app/core/)

### 5.1 安全配置 (security.py)

#### `create_access_token()`
- **函数签名**: `def create_access_token(user_id: str, openid: str) -> str`
- **功能**: 创建 JWT 访问令牌
- **参数**:
  - `user_id`: str - 用户 ID
  - `openid`: str - 微信 openid
- **流程**:
  1. 构建令牌载荷（sub, openid, exp, iat）
  2. 使用 HS256 算法编码 JWT
  3. 返回令牌字符串

#### `verify_token()`
- **函数签名**: `def verify_token(token: str) -> Optional[TokenPayload]`
- **功能**: 验证 JWT 令牌
- **流程**:
  1. 解码 JWT 令牌
  2. 验证过期时间
  3. 返回 TokenPayload 对象

#### `get_current_user()`
- **函数签名**: `async def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(security))`
- **功能**: 获取当前登录用户（依赖注入）
- **用途**: 用于需要认证的接口

### 5.2 应用配置 (config.py)

#### `Settings` (类)
- **功能**: 应用配置类，管理所有配置项
- **配置项**:
  - `APP_NAME`: 应用名称
  - `APP_VERSION`: 应用版本
  - `DEBUG`: 调试模式
  - `HOST`: 服务器主机
  - `PORT`: 服务器端口
  - `SECRET_KEY`: JWT 密钥
  - `ACCESS_TOKEN_EXPIRE_MINUTES`: Token 过期时间
  - `WX_APPID`: 微信小程序 AppID
  - `WX_SECRET`: 微信小程序密钥
  - `ALLOWED_ORIGINS`: CORS 允许来源
  - `DATA_DIR`: 数据存储目录

## 七、登录功能完整调用链

```
前端请求
    ↓
POST /api/v1/auth/wx-login
    ↓
wechat_login() [auth.py]
    ↓
├─ 方式1: 从 Header 读取 X-WX-OPENID（云托管）
└─ 方式2: WechatService.code2session(code)（本地开发）
    ↓
UserService.get_or_create_user(openid, unionid)
    ↓
Database.get_user_by_openid() / Database.create_user()
    ↓
create_access_token(user_id, openid)
    ↓
UserService.has_character()
    ↓
Database.has_character()
    ↓
返回 WXLoginResponse
```
