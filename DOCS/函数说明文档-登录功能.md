## 一、API 接口层 (app/api/)

### 1.1 认证接口 (auth.py)

#### `wechat_login()`
- **函数签名**: `async def wechat_login(login_data, x_wx_openid, x_wx_unionid, x_wx_appid)`
- **路由**: `POST /api/v1/auth/wx-login`
- **功能**: 微信用户登录入口，支持两种登录方式
  - 云托管免鉴权方式（优先）：从 Header 读取微信用户信息
  - 传统 code 方式：使用微信临时凭证换取用户信息
- **流程**:
  1. 优先检查 Header 中的 X-WX-OPENID（云托管方式）
  2. 如无 Header，则使用 code 调用微信 API 换取 openid
  3. 调用 UserService 获取或创建用户（await）
  4. **登录时自动检查角色，如无角色则自动创建（新增）**
  5. 生成 JWT 访问令牌
  6. 返回登录结果
- **重要变更**: 
  - v2.1.0 起，登录时自动创建角色，无需前端判断 has_character
  - has_character 始终返回 true

## 二、数据模型层 (app/models/)

### 2.1 认证模型 (auth.py)

#### `WXLoginRequest`
- **类型**: Pydantic BaseModel
- **功能**: 微信登录请求参数模型
- **字段**:
  - `code`: Optional[str] - 微信登录临时凭证（云托管方式可不传）

#### `WXLoginResponse`
- **类型**: Pydantic BaseModel
- **功能**: 微信登录响应数据模型
- **字段**:
  - `access_token`: str - JWT 访问令牌
  - `token_type`: str - 令牌类型，固定为 "bearer"
  - `expires_in`: int - 令牌过期时间（秒），默认 7200
  - `user_id`: str - 用户唯一标识
  - `has_character`: bool - 用户是否已创建游戏角色（始终为true）

#### `TokenPayload`
- **类型**: Pydantic BaseModel
- **功能**: JWT 令牌载荷模型，定义令牌中包含的数据
- **字段**:
  - `sub`: str - 用户 ID（subject）
  - `openid`: str - 微信用户 openid
  - `exp`: Optional[int] - 过期时间戳

### 2.2 用户模型 (user.py)

#### `UserBase`
- **类型**: Pydantic BaseModel
- **功能**: 用户基础属性模型
- **字段**:
  - `openid`: str - 微信用户唯一标识
  - `unionid`: Optional[str] - 微信 unionid
  - `nickname`: Optional[str] - 用户昵称
  - `avatar_url`: Optional[str] - 用户头像 URL

#### `UserCreate`
- **类型**: Pydantic BaseModel，继承 UserBase
- **功能**: 创建用户请求模型

#### `User`
- **类型**: Pydantic BaseModel，继承 UserBase
- **功能**: 用户完整数据模型
- **字段**:
  - `id`: str - 用户 ID
  - `created_at`: datetime - 创建时间
  - `updated_at`: datetime - 更新时间
  - `last_login_at`: Optional[datetime] - 最后登录时间
  - `is_active`: bool - 是否激活

#### `UserResponse`
- **类型**: Pydantic BaseModel
- **功能**: 用户响应数据模型
- **字段**:
  - `id`: str - 用户 ID
  - `nickname`: Optional[str] - 用户昵称
  - `avatar_url`: Optional[str] - 头像 URL
  - `created_at`: datetime - 创建时间
  - `has_character`: bool - 是否已创建角色（始终为true）

### 2.3 角色模型 (character.py)

#### `CharacterBase`
- **类型**: Pydantic BaseModel
- **功能**: 角色基础属性模型
- **字段**:
  - `nickname`: str - 角色昵称（1-20 字符）
  - `uid`: str - 角色唯一编号（8位数字）
- **变更记录**:
  - v2.1.0: `name` 更名为 `nickname`，新增 `uid` 字段

#### `CharacterCreate`
- **类型**: Pydantic BaseModel
- **功能**: 创建角色请求模型（空模型，无需传入参数）
- **说明**: 角色信息由后端自动生成

#### `Character`
- **类型**: Pydantic BaseModel，继承 CharacterBase
- **功能**: 角色完整数据模型
- **字段**:
  - `id`: str - 角色 ID
  - `user_id`: str - 所属用户 ID
  - `nickname`: str - 角色昵称（默认格式：意识体{uid}）
  - `uid`: str - 角色唯一编号（8位数字）
  - `created_at`: datetime - 创建时间
  - `updated_at`: datetime - 更新时间
- **变更记录**:
  - v2.1.0: 移除 `level` 和 `exp` 字段

#### `CharacterResponse`
- **类型**: Pydantic BaseModel
- **功能**: 角色响应数据模型
- **字段**:
  - `id`: str - 角色 ID
  - `nickname`: str - 角色昵称
  - `uid`: str - 角色唯一编号（8位数字）
  - `created_at`: datetime - 创建时间
- **变更记录**:
  - v2.1.0: 移除 `level` 和 `exp` 字段，新增 `uid` 字段

## 三、服务层 (app/services/)

### 3.1 微信服务 (wechat_service.py)

#### `WechatService` (类)
- **功能**: 微信服务类，封装与微信服务器的交互
- **类属性**:
  - `WX_API_BASE`: str - 微信 API 基础地址

##### `WechatService.code2session()`
- **函数签名**: `async def code2session(cls, code: str) -> Dict[str, Any]`
- **功能**: 微信登录凭证校验，通过 code 换取 openid 和 session_key
- **参数**:
  - `code`: str - 微信登录临时凭证
- **流程**:
  1. 构建请求参数（appid, secret, js_code, grant_type）
  2. 发送 HTTP GET 请求到微信 API
  3. 检查返回结果中的错误码
  4. 返回包含 openid、session_key 等信息的字典

##### `WechatService.get_access_token()`
- **函数签名**: `async def get_access_token(cls) -> str`
- **功能**: 获取微信小程序全局 access_token
- **流程**:
  1. 构建请求参数（grant_type, appid, secret）
  2. 发送 HTTP GET 请求到微信 API
  3. 返回 access_token 字符串

### 3.2 用户服务 (user_service.py)

#### `UserService` (类)
- **功能**: 用户服务类，处理用户业务逻辑，所有方法均为异步实现
- **设计原则**: 与 FastAPI 异步架构保持一致，所有数据库操作使用 await

##### `UserService.get_or_create_user()`
- **函数签名**: `async def get_or_create_user(cls, openid: str, unionid: Optional[str] = None) -> User`
- **功能**: 获取或创建用户
- **参数**:
  - `openid`: str - 微信 openid
  - `unionid`: Optional[str] - 微信 unionid
- **流程**:
  1. 通过 openid 查询用户是否存在（await）
  2. 如存在，更新最后登录时间（await）并返回用户
  3. 如不存在，创建新用户（await）并返回

##### `UserService.get_user_by_id()`
- **函数签名**: `async def get_user_by_id(cls, user_id: str) -> Optional[User]`
- **功能**: 通过用户 ID 获取用户信息
- **参数**:
  - `user_id`: str - 用户ID
- **返回**: 用户对象或None

##### `UserService.has_character()`
- **函数签名**: `async def has_character(cls, user_id: str) -> bool`
- **功能**: 检查用户是否已创建角色
- **参数**:
  - `user_id`: str - 用户ID
- **返回**: 是否已创建角色

##### `UserService.create_character()`
- **函数签名**: `async def create_character(cls, user_id: str) -> Character`
- **功能**: 为用户创建角色
- **参数**:
  - `user_id`: str - 用户ID
- **返回**: 角色对象
- **重要说明**:
  - v2.1.0: 移除 `name` 参数，角色信息由后端自动生成
  - 自动生成8位唯一UID
  - 自动生成昵称"意识体{uid}"

##### `UserService.get_character_by_user()`
- **函数签名**: `async def get_character_by_user(cls, user_id: str) -> Optional[Character]`
- **功能**: 获取用户的角色信息
- **参数**:
  - `user_id`: str - 用户ID
- **返回**: 角色对象或None

#### `create_access_token()`
- **函数签名**: `def create_access_token(user_id: str, openid: str) -> str`
- **功能**: 创建 JWT 访问令牌
- **参数**:
  - `user_id`: str - 用户 ID
  - `openid`: str - 微信 openid
- **流程**:
  1. 构建令牌载荷（sub, openid, exp, iat）
  2. 使用 HS256 算法编码 JWT
  3. 返回令牌字符串

#### `verify_token()`
- **函数签名**: `def verify_token(token: str) -> Optional[TokenPayload]`
- **功能**: 验证 JWT 令牌
- **流程**:
  1. 解码 JWT 令牌
  2. 验证过期时间
  3. 返回 TokenPayload 对象

#### `get_current_user()`
- **函数签名**: `async def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(security))`
- **功能**: 获取当前登录用户（依赖注入）
- **用途**: 用于需要认证的接口

## 九、异步架构说明

### 9.1 为什么全面异步化

**问题背景**:
- 原同步包装类 `WxCloudDatabaseSync` 使用 `asyncio.new_event_loop()` 和 `run_until_complete()`
- FastAPI 本身运行在异步事件循环中
- 在已有事件循环中调用 `run_until_complete()` 会抛出错误：
  `Cannot run the event loop while another loop is running`

**解决方案**:
- 所有服务层方法改为 `async def`
- 所有数据库操作使用 `await`
- 与 FastAPI 异步架构完全兼容

### 9.2 性能优势

- **高并发**: 单进程可处理大量并发请求
- **非阻塞**: IO 操作不阻塞事件循环
- **资源节省**: 协程比线程更轻量
- **易于扩展**: 支持 1000+ 日活用户

### 9.3 调用规范

所有涉及数据库操作的地方必须使用 `await`：

```python
# 正确
user = await UserService.get_or_create_user(openid)
has_char = await UserService.has_character(user.id)

# 错误（旧版本）
user = UserService.get_or_create_user(openid)  # 缺少 await
```
